    # 计算机结构

    • 程序计数器PC：用于存放下一条指令所在单元的地址的地方。在程序执行前，必须将程序的起始地址，即程序的一条指令所在的内存单元地址送入程序计数器，当执行指令时，cpu将自动修改程序计数器的内容，即每执行一条指令程序计数器增加一个量，使其指向下一个待指向的指令。程序的转移等操作也是通过寄存器来实现的。
    • cpu的程序计数器PC用于保存要执行的指令的地址吗，访问内存时，需要将内存地址送入存储器地址寄存器MAR中，向内存写入数据时，待写入的数据要先放入数据寄存器MUR。程序中的指令一般放在内存中，要执行时，首先要访问内存取得指令保存在指令寄存器IR中。计算机中指令的执行过程一般分为取指令、分析指令并获得操作数、运算和传送结果等阶段，每条指令执行时都要经过这几个阶段。
    • 地址寄存器：一般用来保存当前cpu所访问的内存单元的地址，以方便内存的读写操作。
    • 累加器：时专门存放算术或逻辑运算的一个操作数运算和运行结果的寄存器。
    • 算术逻辑单元ALU是cpu的执行单元，主要负责运算工作。

    # 寻址方式

    • 立即寻址：指令的地址字段指出的不是操作数的地址，而是操作数本身。
    • 直接寻址：指令格式的地址字段中直接指出操作数在内存的地址。
    • 寄存器寻址：指令中给出的操作数地址不是内存地址的单元号，而是通用寄存器的编号（当操作数不存放在内存中，而是放在cpu的通用寄存器中时，可采用寄存器的寻址方式）。
    • 寄存器的间接寻址方式与寄存器寻址方式的区别在于：指令格式中操作数地址所指向的寄存器中存放的内容不是操作数而是操作数的地址。
    • 相对寻址的特点：把程序计数器PC的内容加上指令格式中的形式地址来形成操作数的有有效地址。

    # 总线

    • 数据总线是用于在计算机中传送数据的总线，他可以吧cpu的数据传送到存储器或输入输出接口等其他部件，也可以将其他部件的数据传送到cpu。数据总线的位数是微型计算机的一个重要指标，他通常与微处理的字长相一致。
    • 地址总线是传送地址信息的总线，根据地址总线的多少可以确定内存的大小，如32位的地址总线可以允许2的32次方的内存容量

    # pv操作

    • 通俗理解p操作用于阻塞线程，V操作用于唤醒线程。
    • 在前驱图中，有几个前驱，该程序就实现几个P操作，有几个后继就实现几个V操作。
    • 对于前驱图，箭头表示前趋和后继关系，前趋进程完成需要通知后继进程（用V操作通知），后继进程开始前需要检查前趋进程是否完成（用P操作检查）。也就是说，在前趋图中，每一个箭头流出指向后继进程，都会有一个V操作通知后继，每一个箭头的流入都是始于前趋进程，需要检查前趋进程是否完成，用P操作进程检查。
